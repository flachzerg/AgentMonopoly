# 分支决策根因修复报告

## 1. 问题背景

线上出现两类异常表现：

- Agent 日志中频繁出现 `pass`（跳过）动作，回合价值低且行为异常。
- 真人回合在“分支选择”场景易卡住：棋子已移动完成，但仍提示分支决策，`target_tile_id` 变化却不再推进。

## 2. 根因结论

根因不是单点 bug，而是**决策时序设计错误**：

1. 分支路径的实际选择发生在 `_roll_and_settle()` 的掷骰移动过程中（移动前/移动中）。
2. 系统却在移动完成后（`DECISION` 阶段）暴露 `set_route_preference` 动作。
3. 该动作对“本次已完成移动”没有意义，只会造成后置无效操作。
4. 同时 `DECISION` 阶段无条件追加 `pass`，导致 Agent 会把 `pass` 当成合法动作输出。

结论：之前给真人增加 `pass` 只是绕过卡死，不是根修。

## 3. 修复策略

本次按根因修复，核心原则是：

- 不再在移动后暴露“本回合无意义”的分支选择动作。
- 对“无可执行决策动作”的回合自动收尾，不再依赖 `pass` 作为回合终止手段。

## 4. 实施改动

### 4.1 `backend/app/game_engine.py`

- 移除 `set_route_preference` 作为合法动作入口（不再参与回合动作合同）。
- 移除 `DECISION` 阶段中对 `set_route_preference` 的动作生成。
- 移除 `DECISION` 阶段末尾无条件追加 `pass` 的逻辑。
- 调整 `apply_action()` 回合收尾规则：
  - `roll_dice` 后若进入 `DECISION` 且无动作可执行，自动 `_finalize_turn()`；
  - 非 `roll_dice` 的 `DECISION` 动作仍按既有逻辑执行后收尾。
- `human_visible_actions()` 不再暴露 `set_route_preference/pass`。

### 4.2 `backend/app/api/games.py`

- `_run_model_decision_once()` 中，当 `advance_to_decision_if_needed()` 触发了自动收尾并切到下一玩家时，改为返回 `accepted=True`（`turn advanced`），避免被误判为流程阻塞。

### 4.3 测试修正

- `backend/tests/test_game_engine.py`
  - 更新了回合推进断言，兼容“需要决策”与“自动收尾”两种合法结果。
  - 将“分支偏好选项出现”测试改为“后置分支偏好不再暴露”。
- `backend/tests/test_concurrency_simulation.py`
- `backend/tests/load_test_100_games.py`
  - 不再硬编码 `pass`，改为按当前 `allowed_actions` 驱动。

## 5. 验证结果

- `backend/tests/test_game_engine.py`：`7 passed`
- 关键验证点：
  - Agent 不再因为动作白名单而拿到“后置分支 + 跳过”的低价值组合。
  - 真人在“本回合无有效决策动作”时不会卡在 `DECISION`。

## 6. 兼容性与后续建议

- 当前保留了 `route_preference_tile_id` 字段（数据兼容），但不再通过后置动作修改。
- 下一阶段建议：
  - 若需要“分支选择”交互，应在“掷骰移动过程中”引入前置分支决策机制（而非落点后决策）。
  - 将 `branch_decision` 等待语义细分为“移动前分支决策”与“落点后业务决策（事件/联盟等）”。
