# AgentMonopoly 产品与开发待办总览（v2.2）

文档日期：2026-04-18  
适用范围：点开即玩（无注册、无登录）Web 版本

## 1. 当前目标（统一方向）

最终目标不是“能跑 demo”，而是“陌生用户打开网页后，能顺畅开一局、玩一局、看懂一局”。

目标路径：
1. 进入配置页，快速完成房间与玩家配置
2. 进入对局页，用户仅在自己回合做关键动作，其他阶段系统自动推进
3. 对局结束后自动进入复盘页，得到一份通俗中文的全局复盘
4. 复盘结果可直接查看、复制、导出，并用于下一局策略改进

## 2. 当前体验的主要问题

### 2.1 视觉层
- 页面风格单调，缺少“游戏感”和“直播可看性”
- 配置页、对局页、复盘页视觉语言不统一
- 层级、间距、重点信息引导不足，首次用户理解成本偏高

### 2.2 地图层
- 地图表现偏功能化，不够清晰与可记忆
- 格子信息密度与可读性平衡不佳
- 玩家位置、地产归属、关键地块提示不够直观

### 2.3 交互层
- 动作区现在“选项过多”，用户心智负担偏高
- 理想状态应更接近“轮到我时我做决定，其他时间系统自己跑”

### 2.4 复盘层
- 已有日志与事件流，但缺少“人话版全局结论”
- 目前复盘更像数据面板，不像故事化解读
- 还缺少“这一局为什么赢/输、下局怎么改”的清晰结构

### 2.5 Agent 卖点层
- “AI自进化”尚未形成闭环
- 缺少可观察、可验证、可回滚的迭代机制

## 3. v2.2 必做范围（按优先级）

## P0（必须先完成）
1. 全站视觉重构（配置页/对局页/复盘页统一设计系统）
2. 对局主交互改造：
   - 真人回合：只显示“掷骰子 + 必要决策”
   - 非真人回合：默认 AI 自动推进
3. 复盘页新增“中文全局复盘生成”能力（自动触发）
4. 复盘 Prompt 体系落地：输入结构、输出结构、失败降级

## P1（紧跟）
1. 地图重构（信息层级、视觉表现、动效）
2. 关键事件高亮（破产、联盟、大额交易、翻盘点）
3. 复盘页“关键回合卡片 + 时间线联动”

## P2（卖点增强）
1. AI 自进化第一版：
   - 对局日志 -> 策略建议 -> 模板参数更新 -> 下一局生效
2. 自进化效果评估面板：
   - 胜率变化、平均资产变化、非法动作率变化

## 4. 对局交互重定义（重点）

## 4.1 用户动作模型
- 用户仅在“本人回合”进行两个动作：
  1. 掷骰子
  2. 当出现必选分支时做决策（如风险事件、交易同意）

- 其余动作由系统自动推进：
  - AI 决策
  - 结算
  - 日志写入
  - UI 同步

## 4.2 系统状态提示
- 顶部全局状态条需持续显示：
  - 当前回合与当前玩家
  - 当前阶段（等待真人 / AI思考 / 结算中）
  - 自动推进开关状态

## 5. 复盘页目标形态

## 5.1 必备模块
1. 一句话结论（谁赢、核心原因）
2. 三段式分析：前期 / 中期 / 后期
3. 关键转折点（时间线映射到具体回合）
4. 玩家画像（每位玩家 3~5 条）
5. 下局建议（可执行，非空话）

## 5.2 全局复盘 Prompt 设计原则
- 输入材料：
  - 事件时间线
  - Agent 输出日志流
  - 资产变化轨迹
  - 关键动作片段
- 输出要求：
  - 必须中文
  - 通俗易懂
  - 少术语，多解释
  - 每条结论给出“证据回合”

## 5.3 建议输出结构（JSON）
```json
{
  "overview": "string",
  "phase_analysis": [
    {"phase": "前期", "summary": "string", "evidence_turns": [1,2]},
    {"phase": "中期", "summary": "string", "evidence_turns": [8,9]},
    {"phase": "后期", "summary": "string", "evidence_turns": [15,16]}
  ],
  "turning_points": [
    {"turn": 11, "title": "string", "impact": "string"}
  ],
  "player_profiles": [
    {"player_id": "p1", "style": "string", "highlights": ["..."], "issues": ["..."]}
  ],
  "next_game_advice": ["..."]
}
```

## 6. AI 自进化（第一版方案）

## 6.1 闭环定义
1. 采集：每局日志、动作合法性、结局指标
2. 分析：由评估 Agent 生成“模板改进建议”
3. 变更：更新对应席位的策略模板参数（带版本号）
4. 验证：下一批对局对比基线模型
5. 采用或回退：指标变好才生效，否则回退

## 6.2 关键约束
- 参数变更必须可追踪（version、变更说明、时间）
- 支持一键回到上一个稳定版本
- 不直接改动核心规则引擎

## 7. 技术落地任务清单

### 7.1 前端
1. 建立统一 design tokens（色彩、字号、圆角、阴影、间距）
2. 三页面视觉重构与响应式适配
3. 地图组件重构（格子层、玩家层、归属层）
4. 动作区极简化（真人触发 + 其余自动）
5. 复盘页新增 AI 全局复盘展示卡

### 7.2 后端
1. 新增“自动推进状态机”接口与事件
2. 新增“全局复盘生成”任务接口（含失败降级）
3. 新增复盘 Prompt 管理与版本化
4. 新增自进化任务链（分析 -> 建议 -> 参数更新）
5. 新增关键指标 API（对局级、玩家级、模型级）

### 7.3 测试与质量
1. 真人回合/AI自动推进端到端测试
2. 复盘生成稳定性测试（超时、空日志、异常日志）
3. 自进化回退机制测试
4. 前端可用性测试（首局可玩时间、误操作率）

## 8. 建议节奏（两周）

### 第 1 周
- 完成视觉方案定稿
- 对局动作流程改造
- 复盘 Prompt v1 + 接口联调

### 第 2 周
- 地图重构上线
- AI 自进化 v1 跑通
- 指标看板与体验回归

## 9. 完成标准（可上线）

1. 新用户从首页到开局，90 分位 <= 60 秒
2. 一局可完整游玩，流程中断率 <= 1%
3. 对局结束后 10 秒内展示中文复盘
4. 复盘内容可读、可解释，并给出下局建议
5. AI 自进化具备“生效-对比-回退”全链路
