# 程序员A分工文档（引擎与 AI 系统，v2.2）

## 任务清单

1. 对局自动推进主逻辑
- 在非真人回合，系统默认自动推进，不依赖用户点击。
- 到真人回合时暂停，并明确返回“等待真人操作”。

2. 真人回合动作约束
- 真人可操作动作仅保留：`掷骰子`。
- 如遇必须选择分支（例如事件分支），仅返回该分支需要的最小决策选项。
- 其他动作由系统推进，不对真人开放。

3. 复盘生成后端能力
- 对局结束后，基于事件时间线与 Agent 输出日志，触发中文全局复盘生成。
- 产出结构必须稳定，至少包含：
  - 全局结论
  - 分阶段分析（前/中/后）
  - 关键转折点
  - 各玩家表现总结
  - 下一局建议

4. 复盘 Prompt 方案落地
- 明确输入材料拼装方式：事件、日志、关键动作、结果。
- 固化中文提示词模板，输出风格要求“通俗易懂”。
- 输出内容要能直接给普通用户看懂。

5. AI 自进化第一版
- 形成闭环：
  - 局后汇总数据
  - 给出策略改进建议
  - 应用于下一局
- 保留版本标识，确保每次策略变更可追踪。

## 通过标准

1. 自动推进与真人回合切换正确
- 非真人阶段可连续推进。
- 到真人阶段一定暂停，且仅给真人可执行动作。

2. 真人动作入口符合目标
- 真人回合默认只见“掷骰子”。
- 必选分支出现时，只显示该分支需要的决策项。

3. 中文复盘可直接阅读
- 每局结束后可得到一份完整中文复盘。
- 内容不应只是日志罗列，需有结论、原因与建议。

4. 复盘内容有依据
- 复盘中的关键结论能对应到时间线/关键回合。

5. 自进化链路可跑通
- 至少完成一次“局后建议 -> 下一局生效”的闭环演示。
- 可查看当前策略版本标识。
