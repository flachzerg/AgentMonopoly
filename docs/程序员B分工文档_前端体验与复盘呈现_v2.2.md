# 程序员B分工文档（前端体验与复盘呈现，v2.2）

## 任务清单

1. 全页面视觉重做
- 配置页、对局页、复盘页整体视觉全部重做。
- 目标：页面不再单调，风格更像可玩游戏产品。

2. 地图模块重做
- 重新设计地图视觉与信息展示方式。
- 重点提升：位置关系清晰、地产状态清晰、当前局势更直观。

3. 对局页动作区重做
- 动作区默认只保留真人动作：`掷骰子`。
- 仅在必须决策时展示最小决策选项。
- 非真人阶段不让用户做多余操作，界面只展示系统推进状态。

4. 复盘页展示重做
- 接入后端生成的中文全局复盘。
- 同页展示：
  - 事件时间线
  - Agent 输出日志流
  - 中文全局总结
- 让用户在一个页面内看懂“这一局怎么打成这样”。

5. 三页面主流程打磨
- 用户进入先到配置页。
- 配置完成进入新对局页。
- 对局结束进入复盘页。
- 整条路径操作清晰，不让用户迷路。

## 通过标准

1. 视觉体验明显升级
- 三个页面的视觉效果与当前版本有明显差异。
- 页面整体观感不再“单调无趣”。

2. 地图信息可快速读懂
- 用户能直接看出玩家位置、关键地块、地产状态。

3. 动作区符合新交互目标
- 真人回合默认只有“掷骰子”。
- 非真人阶段不会出现可误点动作。

4. 复盘页可直接解释一局对战
- 页面同时包含时间线、日志流、中文全局复盘。
- 用户可读完后明白胜负原因与关键转折。

5. 主流程顺畅
- 按“配置页 -> 对局页 -> 复盘页”完整走通。
- 流程中无卡死、无明显交互歧义。
