## 说明

地图生成引擎（B2-1 ~ B2-8）已独立完成设计，详见：
- `docs/程序员B已完成文档_地图生成引擎设计_v2.2.md`

当前状态补充：

- 因此地图视觉打磨需要并入本文件作为剩余前端美化任务继续推进。

本文件仅保留“未完成”的前端任务，避免任务范围混杂。

## 剩余任务清单

### B1 配置页重做（未完成）

目标：
- 提升配置页可读性与可操作性，支持快速开局和高级配置。

任务：
1. 重做页面布局与视觉层级（房间信息、玩家席位、回合参数）。
2. 增加输入校验与错误提示（game_id、玩家数量、回合上限）。
3. 保持与后端创建对局接口参数一致，不引入前端自定义规则。
4. 预留地图主题选择入口（仅 UI 与参数透传，不在此阶段实现地图渲染切换逻辑）。

验收标准：
- 用户可在 30 秒内完成开局参数配置。
- 配置错误有明确提示，且不会提交无效请求。

### B2 地图视觉二次美化（未完成）

目标：
- 在不改动后端规则链路的前提下，把当前“工程可用地图”升级为“游戏风格地图”。

问题现状：
- 当前底图虽有连线和落点槽位，但仍偏卡片化，游戏氛围不足。
- 地块层级、路径层和局势信息的视觉引导不够强。

任务：
1. 优化地图主题（色彩、纹理、边框、阴影），降低“网页感”。
2. 优化路径连线样式（无箭头前提下增强流动感和方向感）。
3. 优化地块信息密度（名称、类型、价格信息分层更清晰）。
4. 优化棋子落点视觉（多玩家同格时可读且不拥挤）。
5. 增加地图状态高亮规范（当前格、可决策格、关键事件格）。
6. 输出统一视觉 tokens（颜色、间距、字体、圆角）供配置页/对局页/复盘页复用。

验收标准：
- 地图第一眼观感接近“可玩棋盘”，不再像后台管理页面。
- 用户可快速读出玩家位置、关键地块和资产归属。
- 多玩家同格时，棋子显示无明显遮挡歧义。

### B3 对局页动作区重做（未完成）

目标：
- 收敛用户操作，仅展示当前阶段需要的动作，降低误触。

任务：
1. 真人回合默认动作只显示 `掷骰子`。
2. 仅在 `DECISION` 阶段展示必须决策选项。
3. 非真人阶段隐藏可操作按钮，仅展示系统推进状态。
4. 动作面板与日志面板视觉层次优化，保证“状态可读、操作清晰”。

验收标准：
- 非真人阶段不可点击多余动作。
- 真人阶段动作数量符合后端 `allowed_actions` 约束。

### B4 复盘页重做（未完成）

目标：
- 在单页中完成“结果解释 + 过程回看 + 关键转折定位”。

任务：
1. 接入后端中文全局复盘文本。
2. 同页展示事件时间线、Agent 输出日志流、中文总结卡片。
3. 提供关键回合跳转（从总结条目跳转到对应事件或回合位置）。
4. 明确展示胜负原因与关键转折点。

验收标准：
- 用户读完复盘页可明确回答“为什么输赢”。
- 时间线、日志流、总结三者信息一致。

### B5 三页面主流程打磨（未完成）

目标：
- 保证从配置到对局到复盘的路径连贯、无迷路感。

任务：
1. 路由链路固定为：`/setup -> /game/:gameId -> /replay/:gameId`。
2. 每页提供清晰的主操作按钮与返回路径。
3. 对局结束自动跳转复盘页，异常时给出可恢复提示。
4. 处理刷新和直达链接的状态恢复。

验收标准：
- 主流程可完整走通，无卡死和明显交互歧义。
- 刷新页面后可恢复当前局面或给出明确恢复操作。

- 不在本任务中引入新的状态管理框架。
