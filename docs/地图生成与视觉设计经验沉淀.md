# 地图生成与视觉设计经验沉淀

本文档记录了从“单环地图”向“分支图、复杂交叉图”演进过程中，关于坐标布局、连线渲染与视觉调优的试错经验与核心结论，供后续开发地图和调试 UI 参考。

## 1. 物理坐标与重叠控制

**踩坑：** 
最初期望使用前端缩放算法（全局基于密度的宽高缩放 `_scale_for_density`）来防止文字溢出或地块重叠。但当出现复杂分支或回退路线时，算法粗暴压缩地块会导致文字坐标错乱、地块比例失调，甚至出现回退节点直接盖在主线节点上方。

**最佳实践：**
- **禁用全局压缩算法**：保持地块 `w` 和 `h` 绝对大小不变（本项目统一修改为正方形 `100x100`）。
- **显式网格化布局（Grid-based Mapping）**：与其依赖代码计算自动布局避让，不如在 JSON 生成脚本中通过清晰的矩阵/网格进行绝对坐标规划。
- 只有底层的坐标彻底避免了交叉，地图的最终呈现才能清晰稳定。

## 2. 连线附着点与多线重叠消除

**踩坑：**
早期连线的始末点默认设置为地块的“正中心”，这不仅导致连线穿模经过地块内容，还在多条分支从同一个地块散开时，导致多条贝塞尔曲线的根部完全重叠在一起，无法分辨。

**最佳实践：**
1. **边缘中点引出**：计算两地块相对位置（如目标在右，则从右侧边缘中点出，连接到左侧边缘中点入）。
2. **多线等分偏移**：对于同一边出现多条连线（比如 `T03` 的底部同时连出 `T04` 和 `C01`），算法必须预先统计该边连线的数量，并通过偏移量（offset）将这些线的起点在边缘上**均匀错开**。这彻底消除了线根部的重叠。

## 3. 贝塞尔曲线的优雅运用

**踩坑：**
使用二次贝塞尔（Quadratic Bezier, `Q`）或者控制点设置不当时，在处理对角线或斜向跨度较大的节点时，会产生生硬的“对角斜线”或“锐角折弯”，毫无美感可言。

**最佳实践：**
- **采用三次贝塞尔（Cubic Bezier, `C`）**。
- **定向拉伸控制点**：如果连线是从右侧出、左侧入，那么它的两个控制点应该分别在出发点和结束点的**水平向外**拉伸一定的距离（如 `dist * 0.45`）。
- **大跨度不对称布局**：要最大化展示贝塞尔的柔美，地图布局（如 `bezier_showcase`）必须刻意使用“不对称/交错”的绝对坐标，搭配绝对**中心对称**的整体大架构，让曲线在长距离跨越时自然形成极具工业美感的 "S" 形。

## 4. 渐进式披露与对齐

- **地块内文本**：为了防止拥挤，抛弃显示繁杂的 price、toll，仅保留“地块类型”与“地块名称”。
- **垂直对齐**：使用相对百分比（如 `h * 0.42` 和 `h * 0.62`）悬挂文本，以完美适应正方形地块。
- **信息转移**：拥有者（owner）的表达从文本转移为颜色的变化。
- **棋子槽位**：从一字排开改为分布在地块内边距的四个角上，最大程度留出中心文字空间。

## 5. 核心修改脚本参考
- `scripts/generate_new_maps.py`：负责常规与复杂网格地图（大单环、大分支、复杂交叉）的坐标建模。
- `scripts/generate_bezier.py`：专门用于设计展示完美 S 曲线的对称性节点架构。
- `backend/app/map_svg.py`：包含了所有上述关于边缘计算、贝塞尔生成与等分防重叠的核心渲染逻辑。
